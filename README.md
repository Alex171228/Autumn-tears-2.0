---

## 1. Введение

Наименование разработки: веб-приложение для расчета, моделирования и визуализации траекторий роботов-манипуляторов `Autumn-tears-2.0`.

Краткая характеристика области применения:
- моделирование движения промышленных роботов-манипуляторов;
- расчет траекторий, рабочих областей и связанных графиков;
- учебное, исследовательское и инженерное применение в задачах автоматизации и робототехники.

Объект использования:
- вычислительные системы с современным веб-браузером;
- серверные и пользовательские рабочие станции под управлением Windows, Linux и macOS;
- контейнерная среда выполнения Docker.

Программа используется как веб-система с микросервисной архитектурой, в которой:
- Go-сервисы обеспечивают маршрутизацию, авторизацию, администрирование и работу с конфигурациями;
- Python-сервис обеспечивает расчетное ядро, построение графиков и расчет рабочей области.

## 4. Требования к программе или программному изделию

### 4.1. Требования к функциональным характеристикам

#### 4.1.1. Состав выполняемых функций

Программа должна обеспечивать выполнение следующих функций:

1. Расчетные функции:
- позиционное управление по циклограмме;
- контурное управление;
- расчет траекторий и производных параметров;
- расчет рабочей области;
- формирование графиков и массивов данных.

2. Функции конфигурирования:
- задание типа робота;
- настройка ограничений, параметров звеньев и приводов;
- настройка ПИД-регуляторов;
- сохранение конфигурации пользователя;
- импорт и экспорт конфигурации в текстовом формате.

3. Функции доступа:
- регистрация пользователя;
- аутентификация пользователя;
- смена пароля;
- получение данных текущего пользователя.

4. Административные функции:
- просмотр пользователей;
- управление правами администратора;
- просмотр и удаление пользовательских конфигураций;
- удаление пользователей.

5. Инфраструктурные функции:
- маршрутизация запросов через единый gateway;
- хранение конфигураций в MongoDB;
- хранение расчетных сессий в Redis;
- контейнерный запуск через Docker Compose.

#### 4.1.2. Организация входных данных

Входные данные программы включают:
- JSON-запросы на настройку параметров робота;
- JSON-запросы на запуск расчетов и получение графиков;
- текстовые файлы конфигураций робота;
- учетные данные пользователей;
- административные команды управления пользователями и конфигурациями.

#### 4.1.3. Организация выходных данных

Выходные данные программы включают:
- JSON-ответы API;
- графики в виде base64-представления изображений;
- массивы данных траекторий и производных характеристик;
- текстовые файлы конфигураций;
- сообщения о состоянии операций и ошибках.

#### 4.1.4. Временные характеристики

Программа должна обеспечивать:
- отклик API для типовых CRUD-запросов в пределах, приемлемых для интерактивной веб-работы;
- расчет траектории и подготовку графиков в пределах, приемлемых для пользовательского сценария без длительного ожидания;
- запуск и восстановление контейнерного стенда в пределах стандартного времени `docker compose up`.

Целевые ориентиры:
- типовые API-запросы: до 1 секунды;
- типовой расчет траектории: до 5 секунд;
- получение интерфейса после запуска стека: до нескольких секунд после готовности контейнеров.

### 4.2. Требования к надежности

Программа должна обеспечивать:
- устойчивую обработку некорректных входных данных;
- сохранение работоспособности при ошибках отдельных пользовательских запросов;
- возможность повторного запуска микросервисов без ручного восстановления данных в MongoDB и Redis;
- логирование ошибок сервисов;
- изоляцию расчетного состояния по сессиям.

Требования к контролю входной и выходной информации:
- валидация входных JSON-структур;
- проверка корректности идентификаторов и параметров;
