# ТЕХНИЧЕСКОЕ ЗАДАНИЕ

## На разработку веб-приложения для моделирования роботов-манипуляторов

---

## 1. ВВЕДЕНИЕ

Веб-приложение для расчёта и визуализации траекторий промышленных роботов-манипуляторов представляет собой современную веб-версию десктопного приложения, разработанного на PyQt5.

**Область применения:** Моделирование и расчёт траекторий движения промышленных роботов-манипуляторов для научных исследований, обучения и проектирования систем автоматизации.

**Объект использования:** Вычислительные системы с веб-браузером, работающие под управлением операционных систем Windows, Linux, macOS.

**Репозиторий проекта:** [https://github.com/Alex171228/Autumn-tears-2.0](https://github.com/Alex171228/Autumn-tears-2.0)

## 3. НАЗНАЧЕНИЕ РАЗРАБОТКИ

### 3.1. Функциональное назначение

Веб-приложение предназначено для:

- Расчёта траекторий движения промышленных роботов-манипуляторов четырёх типов:
  - Декартовый робот — перемещение по трём взаимно перпендикулярным осям X, Y, Z
  - Цилиндрический робот — использует цилиндрическую систему координат
  - SCARA робот — горизонтально-шарнирный робот для сборочных операций
  - Колер — аналогичен цилиндрическому роботу, специализированная кинематика

- Визуализации результатов расчётов в виде графиков:
  - Траектории движения (обобщённые и декартовы координаты)
  - Скорости и ускорения звеньев
  - Электрические параметры (напряжение, ток)
  - Механические параметры (моменты двигателя и нагрузки)
  - Ошибки регулирования

- Настройки параметров робота:
  - Параметры ПИД-регулятора
  - Параметры электродвигателей
  - Конструктивные параметры робота
  - Ограничения рабочей области

- Управления конфигурациями:
  - Сохранение и загрузка конфигураций локально (файл)
  - Облачное хранение конфигураций (для авторизованных пользователей)

- Администрирования системы:
  - Управление пользователями
  - Управление сохранёнными конфигурациями

### 3.2. Эксплуатационное назначение

Приложение предназначено для использования в образовательных, научно-исследовательских и проектных организациях, занимающихся разработкой и изучением систем управления промышленными роботами-манипуляторами.

## 4. ТРЕБОВАНИЯ К ПРОГРАММЕ ИЛИ ПРОГРАММНОМУ ИЗДЕЛИЮ

### 4.1. Требования к функциональным характеристикам

#### 4.1.1. Состав выполняемых функций

Программа должна обеспечивать выполнение следующих функций:

1. **Расчёт траекторий:**
   - Позиционное управление с заданием циклограммы движения
   - Контурное управление (линейное и круговое)
   - Сплайн-интерполяция траекторий кубическими сплайнами

2. **Визуализация результатов:**
   - Построение графиков траекторий в различных системах координат
   - Отображение графиков скоростей и ускорений
   - Визуализация электрических и механических параметров
   - Отображение рабочей области робота

3. **Управление конфигурациями:**
   - Сохранение конфигурации в файл формата .txt
   - Загрузка конфигурации из файла
   - Облачное сохранение и загрузка конфигураций

4. **Система авторизации:**
   - Регистрация новых пользователей
   - Авторизация пользователей
   - Смена пароля
   - Управление пользователями (для администраторов)

5. **Администрирование:**
   - Просмотр списка пользователей
   - Назначение/снятие прав администратора
   - Удаление пользователей
   - Управление конфигурациями всех пользователей

#### 4.1.2. Организация входных данных

Входные данные программы включают:

- Параметры робота (тип, конструктивные параметры, ограничения)
- Параметры ПИД-регулятора (коэффициенты Kp, Ki, Kd для каждого звена)
- Параметры двигателей (моменты инерции, постоянные времени, максимальные напряжения и др.)
- Циклограмма движения (массивы времени и координат)
- Параметры контурного управления (координаты точек, радиусы, скорости)
- Учётные данные пользователя (для авторизованного доступа)

#### 4.1.3. Организация выходных данных

Выходные данные программы включают:

- Графики траекторий в формате изображений (base64)
- Массивы данных для построения графиков на клиенте
- Файлы конфигурации в формате .txt
- JSON-ответы API для взаимодействия с клиентской частью

#### 4.1.4. Временные характеристики

- Время расчёта траектории: не более 5 секунд для типичных конфигураций
- Время отклика API: не более 1 секунды для стандартных запросов
- Время загрузки интерфейса: не более 3 секунд при стандартном подключении

### 4.2. Требования к надёжности

#### 4.2.1. Обеспечение устойчивого функционирования

- Программа должна корректно обрабатывать некорректные входные данные с выдачей понятных сообщений об ошибках
- При сбоях в работе программы должна обеспечиваться возможность восстановления данных из сохранённых конфигураций
- Время восстановления после отказа: не более 30 секунд (перезапуск сервера)

#### 4.2.2. Контроль входной и выходной информации

- Валидация всех входных данных на стороне сервера
- Проверка типов данных и диапазонов значений
- Защита от SQL-инъекций через использование ORM (SQLAlchemy)
- Проверка формата файлов при загрузке конфигураций

#### 4.2.3. Обработка ошибок

- Все ошибки должны логироваться на сервере
- Пользователю должны предоставляться понятные сообщения об ошибках
- Критические ошибки не должны приводить к полной остановке работы приложения

### 4.3. Условия эксплуатации

#### 4.3.1. Условия эксплуатации технических средств

#### 4.3.2. Вид обслуживания

- Обслуживание: пользовательское

### Б.4.1. Настройка переменных окружения (опционально)

Для работы виджета погоды и безопасности приложения создайте файл `.env` в корне проекта.

**Важно о SECRET_KEY:**

`SECRET_KEY` — это секретный ключ для подписи и проверки JWT токенов авторизации. Он используется для:
- Подписи токенов при входе пользователей
- Проверки подлинности токенов при каждом запросе
- Защиты от подделки токенов

**⚠️ КРИТИЧЕСКИ ВАЖНО:** 
- **НЕ используйте** значение по умолчанию `your-secret-key-change-in-production` в продакшене!
- Используйте **случайный, длинный и уникальный** ключ
- **Никогда не публикуйте** этот ключ в репозитории (файл `.env` уже в `.gitignore`)

**Генерация безопасного SECRET_KEY:**

**Windows (PowerShell):**
```powershell
python -c "import secrets; print('SECRET_KEY=' + secrets.token_urlsafe(32))"
```

**Linux/Mac:**
```bash
# Способ 1: через Python
python3 -c "import secrets; print('SECRET_KEY=' + secrets.token_urlsafe(32))"

# Способ 2: через OpenSSL
openssl rand -hex 32
```

Скопируйте сгенерированный ключ и замените `your-secret-key-change-in-production` в файле `.env`.

> **Примечание:** Виджет погоды будет работать только при наличии `OPENWEATHER_API_KEY`. Остальные функции приложения работают без этого файла, но для безопасности рекомендуется установить `SECRET_KEY`.

## ПРИЛОЖЕНИЕ В. Использование

### В.1. Быстрый старт

1. Откройте http://127.0.0.1:5173/ в браузере
2. Выберите тип робота: **Режим → Тип робота**
3. Настройте параметры в меню **Параметры**:  
   - Движение → Позиционное/Контурное  
   - Регулятор (коэффициенты ПИД)  
   - Двигатель (параметры электродвигателей)  
   - Конструктивные параметры робота
4. Задайте циклограмму движения: **Параметры → Движение → Позиционное → Циклограмма**
5. Нажмите **Расчет → Рассчитать траекторию**
6. Просмотрите результаты на вкладках:  
   - **Траектория** — графики различных параметров  
   - **Рабочая область** — визуализация рабочего пространства робота  
   - **Лог** — журнал операций

### В.2. Работа с файлами

- **Данные → Файл → Загрузить из файла** — загрузка конфигурации из .txt файла
- **Данные → Файл → Скачать в файл** — сохранение конфигурации в файл

### В.3. Облачное хранение (требуется авторизация)

1. Зарегистрируйтесь: **Авторизация → Регистрация**
2. Войдите в аккаунт: **Авторизация → Войти**
3. Сохраните конфигурацию: **Данные → Облако → Сохранить**
4. Загрузите сохранённую: **Данные → Облако → Мои конфигурации**

### В.4. Администрирование

По умолчанию создаётся администратор:

- **Логин:** admin
- **Пароль:** admin123

**Важно:** Смените пароль администратора после первого входа!

Функции админ-панели (**Админ → Управление пользователями**):

- Просмотр списка пользователей
- Назначение/снятие прав администратора
- Удаление пользователей
- Просмотр и загрузка любых конфигураций
- Удаление конфигураций

## ПРИЛОЖЕНИЕ Г. API Endpoints

### Г.1. Авторизация

- `POST /api/register` — регистрация
- `POST /api/login` — вход
- `GET /api/me` — информация о текущем пользователе
- `POST /api/change-password` — смена пароля

### Г.2. Робот

- `POST /api/robot/configure` — настройка параметров
- `POST /api/robot/calculate` — расчёт траектории
- `GET /api/robot/plot/{type}` — получение графика
- `GET /api/robot/workspace` — рабочая область
- `GET /api/robot/spline-cyclegram` — данные сплайна

### Г.3. Конфигурации

- `GET /api/configs` — список конфигураций пользователя
- `POST /api/configs` — создание конфигурации
- `PUT /api/configs/{id}` — обновление
- `DELETE /api/configs/{id}` — удаление

### Г.4. Файлы

- `POST /api/data/upload` — загрузка файла
- `POST /api/data/download` — скачивание файла

### Г.5. Админ

- `GET /api/admin/users` — список пользователей
- `GET /api/admin/configs` — все конфигурации
- `DELETE /api/admin/users/{id}` — удаление пользователя
- `PUT /api/admin/users/{id}/toggle-admin` — переключение прав админа
